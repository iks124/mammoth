# COST: Continual Optimization with Symmetry Teleportation for Multi-Task Learning

**arXiv:** 2503.04046  
**Authors:** Zhipeng Zhou, Ziqiao Meng, Pengcheng Wu, Peilin Zhao, Chunyan Miao  
**Venue:** arXiv 2025-03-06  
**Tags:** multi-task learning, symmetry teleportation  

---

## 核心问题

MTL 中的梯度冲突（gradient conflict）和任务不均衡（task imbalance），传统方法通过重加权 loss 或梯度解决，COST 从 symmetry teleportation 角度切入。

---

## 关键技术

### 1. LoRA 作为 Teleportation 机制

传统 symmetry teleportation（Zhao et al. 2022）要求激活函数双射（bijective），对 ReLU 不适用，且计算代价大（逐层伪逆）。

COST 用 LoRA 替代：
```
θ' = θ + Δθ，Δθ = BA（B: d×r, A: r×d）
```
训练完成后 merge 回 backbone，推理时没有额外参数。

### 2. 双目标优化

**目标函数：** `L_lora = L_t - γ · L_g`

**L_t（Loss Invariance）：** 最小化 teleportation 后 loss 变化
```
L_t = (1/K) Σ_i |L_i - L*_i|
```
其中 L*_i 是 teleportation 开始前的 loss（冻结，不更新）。

**L_g（Gradient Maximization / Sharpness）：** 最大化目标点的梯度范数（用 sharpness 估计，避免计算 Hessian）
```
L_g = max_{j} { (1/K) Σ_i R_i · L_i(θ + Δθ + ε_j) }，ε_j ~ Sphere(δ)，j=1..ñ
```
R 是基于任务梯度范数比值的重加权因子（缓解 imbalance）。

### 3. 触发条件

"Dominated Conflict"触发（超过 K/2 个任务梯度与均值梯度冲突）：
```
Σ_i 1[cos(g_i, g_0) < 0] ≥ K/2
```

### 4. Historical Trajectory Reuse (HTR)

Teleportation 后 Adam 的动量状态与新参数不匹配，COST 根据 teleportation 方向与旧梯度的对齐度来调制 momentum：
```
σ = cos_sim(Δθ, g')
v_t = σ β_1 v_{t-1} + (1 - σβ_1) g_t
s_t = σ β_2 s_{t-1} + (1 - σβ_2) g_t²
```
之后重置 σ=1（标准 Adam）。

## 实验结果

- NYUv2（3任务）: COST + FairGrad → Δm% -5.39%（FairGrad: -4.66%，改善 +0.73%）
- CityScapes（2任务）: Δm% 4.30%（FairGrad: 5.18%，改善 +0.88%）
- CelebA（40任务）: MR 4.80（FairGrad-R: 6.35）
- QM9（11任务）: MR 4.18（FairGrad-R: 4.82）
- Plug-and-play：CAGrad/Nash-MTL/FairGrad 均有 +0.88~3.21% Δm% 改善

## 对 CL 的借鉴意义

| COST（MTL）| CL 对应关系 |
|---|---|
| K 个任务同时训练 | 任务 0..t-1 的 memory + 任务 t |
| 梯度冲突触发 | 任务切换时触发 |
| loss invariance on K tasks | loss invariance on old task memory |
| gradient maximization for all | 最大化新任务 t 的梯度 |
| HTR：调制 Adam momentum | 任务 t 开始训练前调制 momentum |

**核心假设**：在任务 t 训练前，用 LoRA 找一个对旧任务 loss 不变、对新任务梯度范数更大的参数点，有利于任务 t 的训练且减少对旧任务的遗忘。

## 已知局限

- 需要 LoRA 支持的架构（PEFT package），对只有 2 层的图模型效果差
- γ 超参数需要调整
- 训练时略有开销（teleportation 频率由冲突条件控制）
