Teleportation + Continual Learning: 系统性失败分析

基于两篇相关文献（COST, B00ST）的理论分析 + 6组实验的综合报告 · 2026-04-14
报告摘要：通过对 COST（MTL）和 B00ST（DLTR）两篇成功论文的理论分析，以及我们6组 CL 实验的系统性失败，本报告得出以下核心结论：

Symmetry Teleportation 解决的是"并发梯度冲突"问题，而 Continual Learning 的遗忘问题是"序列化数据访问"问题——两者是本质不同的问题，teleportation 工具集无法解决 CL 遗忘。

1. 两篇文献如何使用 Teleportation

1.1 COST (arXiv 2503.04046) — Multi-Task Learning

问题：MTL 中多任务梯度冲突（gradient conflict）和任务不均衡（task imbalance）
冲突定义：超过 K/2 个任务梯度与平均梯度方向相反（cos_sim < 0）
θ' = θ + Δθ，Δθ = BA（LoRA，rank r）

L_t = (1/K) Σ_i |L_i - L*_i| ← loss invariance on ALL K tasks
L_g = max_j { (1/K) Σ_i R_i · L_i(θ + Δθ + ε_j) } ← sharpness proxy
L_lora = L_t - γ · L_g
**关键机制**： - L_t 保证 teleportation 不破坏任何任务的当前 loss（等价于找到 loss-invariant 点） - L_g 最大化梯度范数（sharpness）—— 希望在新点上梯度更丰富，有利于后续优化 - HTR：teleportation 后根据方向对齐度决定是否复用 Adam 动量 - **触发时机**：训练中检测到冲突时立即触发，然后继续正常训练

1.2 B00ST (MM '25) — Deep Long-Tailed Recognition

问题：DLTR 中头部类别梯度压制尾部类别梯度
冲突定义：任意两类别组 cos_sim < 0
θ' = θ + Δθ，Δθ = BA（LoRA）

L_t = Σ_{i=1}^{M} |L_orig^(i) - L_cur^(i)| ← loss invariance on ALL M groups
L_g = Σ_{i=1}^{M} R_i · ||E[∇h^(i)(θ+rv)]q/r||² ← 加权梯度范数，尾部权重更大
R = M · softmax( (Σ ||g_j|| / ||g_i||)_{i=1}^{M} )

L = L_t - α * L_g
B00ST 改进点（相比 COST）：L_g 不用随机扰动估计 sharpness，而是直接估计梯度范数，且对尾部类别（小梯度组）加重权重——这使 teleportation 优先提升尾部类别的梯度，解决类别不均衡。

3. 为什么 Teleportation 在 CL 中系统性失败

核心理论结论

Symmetry Teleportation 解决的是同一优化步骤内的梯度几何问题（找到梯度更均衡的等价参数点）。Continual Learning 的遗忘问题是跨时间的数据访问问题（旧任务数据不再可用）。两者是本质不同的问题。

3.1 Teleportation 的工作原理（为什么在 MTL/DLTR 有效）

Teleportation 的理论保证来自 loss-invariant 集合 A：

∀a ∈ A, ∀θ ∈ R^m: L(θ) = L(a · θ)

在这个集合中找梯度范数最大的点：

θ* = argmax_{a∈A} ||∇L(a·θ)||²

这在 MTL/DLTR 中有效，因为：

  1. Loss invariance 是真实的：teleportation 期间所有数据都在，L_t 可以在完整数据集上计算，保证不破坏任何任务/类别的 loss
  2. 梯度几何在新点更好：sharpness/gradient norm 更大意味着后续梯度步骤更有效
  3. 在线恢复：即使 LoRA merge 引入微小扰动，后续正常训练会立刻校正

3.2 在 CL 中失败的三个根本原因

原因1：Loss Invariance 是假的

我们的 L_t 只能在 ER buffer（500样本）上计算。
这500个样本只代表旧任务的一小部分，
buffer 上的 loss 不变 ≠ 旧任务完整 loss 不变。
伪约束

原因2：无在线恢复机制

teleportation 后，只在新任务上继续训练。
任何对旧任务表征的损坏无法被校正。
H5 seed=3 下降 9.5% 正是因为 LoRA 步骤
干扰了旧任务的 representations。
永久损坏

原因3：问题不同

COST/B00ST 解决的是：
"同一 batch 内，谁的梯度更大？"
CL 遗忘的原因是：
"旧任务的参数空间被新任务覆盖"
梯度几何改变无法解决访问问题。
错误工具

3.3 "自由度 vs 输出不变性"的不可能三角

在 CL 设定下，任何 teleportation 方案都面临以下三角矛盾：

方案自由度输出不变性CL 安全性
ReLU scaling 小（512维） 严格保证 随机（效果微弱）
LoRA + cos_sim（H1b） 大（rank×d×层数） 无保证（delta_norm=3） 破坏旧任务（-3.7%）
LoRA repair（H5） 中（steps=5限制） 部分（buffer上近似） 高方差（std=4.49%）
H4（理论上） 受约束的LoRA buffer近似 预测：≈0，不能改善
不可能三角：在 CL 中，想要"足够大的自由度来改变梯度几何"与"保证不破坏旧任务表征"是互相矛盾的，因为后者需要完整的旧任务数据，而 CL 设定下这数据不存在。

4. B00ST 成功的充分条件分析

B00ST 在 DLTR 上取得 SOTA 的原因不是"teleportation 更强"，而是其应用场景恰好满足了所有前提条件：

  1. 梯度冲突是静态的：头/尾类别的不均衡在整个训练过程中持续存在，teleportation 在每次检测到冲突时都能带来稳定收益
  2. 所有数据始终可访问：L_t 可以在完整数据集上计算，loss invariance 保证是真实的
  3. LoRA merge 后正常训练可以恢复：即使 merge 引入微小扰动，后续 batch（包含所有类别）会立刻校正
  4. 重加权 R_i 针对性解决尾部问题：B00ST 的关键创新是给尾部类别的梯度更大权重，这在 DLTR 中是有针对性的改进（COST 没有这个）

这四个条件在 CL 中全部不满足：

  • 遗忘是动态的（随任务推进逐渐累积）
  • 旧任务数据仅有 ~500 个 buffer 样本
  • LoRA merge 后只训练新任务，无法恢复旧任务损坏
  • 没有等价的"重加权"目标来对抗遗忘

5. 这个 Negative Result 的价值

我们系统性地测试了6种将 symmetry teleportation 迁移到 CL 的方案，全部失败。这不是随机失败——每次失败都有一致的理论解释，指向同一个根本矛盾。

这个 negative result 具有以下学术价值：

  1. 理论厘清：明确区分了"梯度冲突问题"（teleportation 擅长解决）和"序列化遗忘问题"（teleportation 不能解决），防止后续研究者重复这条死路
  2. 边界划定：给出了 symmetry teleportation 成功的充分条件（全数据可访问 + 在线恢复机制），这些条件在原论文中都是隐含的，我们的工作使其显式化
  3. 实证支撑：6种不同参数化方式（ReLU scaling、LoRA+cos_sim、LoRA+sharpness、LoRA repair）覆盖了大部分合理的迁移策略，实证结果全面
  4. 反例贡献：B00ST 和 COST 的作者在论文中没有讨论 CL 的适用性，我们的工作填补了这个空白
